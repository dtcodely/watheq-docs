# مرجع الواجهة البرمجية (API Reference)

جميع مسارات الـ API للواجهة الخلفية (Backend) تبدأ بالمسار الأساسي `/api`. 
ويتطلب استخدام نظام الصلاحيات تمرير رمز مصادقة (JWT) صالح لمعظم الطلبات في ترويسة الطلب.

```http
Authorization: Bearer <token>
```

![اقتراح صورة: مخطط يوضح عملية المصادقة باستخدام JWT وتدفق البيانات بين العميل والخادم]

---

## 1. المصادقة وإدارة الحسابات (`/api/auth`)

| المسار (Endpoint) | الطريقة (Method) | الوصف |
|---|---|---|
| `/register` | POST | إضافة مستخدم جديد للنظام (متاحة للمدراء وأصحاب الصلاحية العُليا فقط) |
| `/login` | POST | تقديم بيانات الاعتماد وتسجيل الدخول، يقوم الخادم بإرجاع الـ JWT |
| `/me` | GET | جلب تفاصيل وبيانات حساب المستخدم الحالي المسجل دخوله |

---

## 2. إدارة الوثائق (`/api/documents`)

المسار الرئيسي لعمليات عرض وإدارة جميع الوثائق في النظام:

| المسار (Endpoint) | الطريقة (Method) | الوصف |
|---|---|---|
| `/` | GET | جلب قائمة الوثائق. تدعم هذه النقطة معلمات تصفية مرنة (مثل: search, fullText, type, status, departmentId) للمقاطع المفلترة والبحث السريع. |
| `/` | POST | إنشاء وثيقة جديدة في النظام. يُستخدم (FormData) إذا تضمن الطلب رفع ملفات ومرفقات مع الوثيقة. |
| `/:id` | GET | جلب التفاصيل الكاملة والمترابطة الخاصة بوثيقة معينة (حسب معرفها). |
| `/:id` | PATCH | تحديث بيانات الوثيقة (مثل تغيير حالة الوثيقة، تعديل رقم التتبع، أو تحويل القسم). |
| `/:id` | DELETE | حذف وثيقة بشكل نهائي (حسب سياسة الحذف في النظام). |

---

## 3. التخلص الآمن والإتلاف (`/api/disposals`)

مسارات متخصصة بخطوات نظام الإتلاف المرحلي والآمن:

| المسار (Endpoint) | الطريقة (Method) | الوصف |
|---|---|---|
| `/` | POST | إرسال طلب إتلاف للوثائق المحددة المنتهية الصلاحية (ينشئ طلب سجل أو أكشن DisposalAction). |
| `/:id/certify` | POST | إنشاء شهادة إتلاف بصيغة PDF نهائية، مدعمة بختم زمني، وتاريخ موثق، وتوقيع النظام لضمان الامتثال القانوني. |

---

## 4. نظام البحث المتقدم ذو النص الكامل (Advanced FTS)

يمكن تفعيل البحث الشامل والدقيق في المحتوى النصي المستخرج عن طريق تقنية الـ OCR، يتم ذلك بتمرير مفاتيح الاستعلام (Queries) التالية في مسار جلب الوثائق `GET /api/documents`:

- `search`: الكلمة، العبارة، أو الجملة المراد البحث عنها في محتويات النظام.
- `fullText=true`: مفتاح اختياري يُفعِّل البحث المكثف داخل حقل `textContent` المستخرج.

**مثال للرد المتفوق المدعوم باقتطاع النص (Snippet Generator):**
يُرجع الخادم الرد مضافاً إليه السياق المحيط بالكلمة المحددة بشكل ذكي.
```json
{
  "data": [
    {
      "id": "123",
      "subject": "عقد توريد أجهزة",
      "textContent": "... بناء على الاتفاق، عقد توريد أجهزة تم تجهيزه وتصديقه من رئيس القسم ...",
      "snippet": "... بناء على الاتفاق، عقد توريد أجهزة تم تجهيزه وتصديقه ..." 
    }
  ],
  "total": 1
}
```
