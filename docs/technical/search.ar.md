# توثيق نظام البحث المتقدم (Advanced Search System)

تم بناء نظام واثق ليحتوي على محرك بحث معزز يتيح للمستخدمين العثور على الوثائق والمستندات بدقة وسرعة بغض النظر عن ضخامة البيانات في النظام. 

![اقتراح صورة: رسم توضيحي لعجلة التروس تمثل محرك البحث أو لقطة شاشة للبحث المتقدم من النظام المستضاف]

## المفاهيم والمكونات الأساسية

1.  **الفهرسة النصية الكاملة (Full-Text Search FTS):** تعتمد على ملحقة `PG_TRGM` (PostgreSQL Trigram) ومؤشرات `GIN Index` لتسريع البحث عبر النصوص الطويلة.
2.  **استخراج النصوص التلقائي (OCR / Text Extraction):** عملية تُجرى في الخلفية (Background Job) عند رفع أي مرفق (Word, PDF, PNG/JPG) وتحويله إلى نص يُخزن بحقل `textContent`.
3.  **الفلترة المتقدمة (Advanced Filtering):** تركيبات مرنة تدمج الكلمات المفتاحية مع معاملات أخرى (כالتاريخ، القسم، النوع، حالة الوثيقة).
4.  **تظليل وإبراز النتائج (Highlighting & Snippeting):** استخراج النص المحيط بالكلمة المبحوثة (Snippet) وتظليلها بلون مميز في جدول النتائج في الواجهة الأمامية.

---

## 1. الفهرسة في قواعد البيانات (Database Indexing)

يُعتبر تفعيل الـ `GIN Index` خطوة محورية. يعمل الـ Trigram على فهرسة النصوص ليس ككلمات مستقلة بل كسلاسل حروف متجاورة (n-grams)، مما يدعم ميزة البحث الجُزئي (Fuzzy Search) والتسامح مع الأخطاء الإملائية، ويدعم اللغة العربية بامتياز.

```sql
CREATE EXTENSION IF NOT EXISTS pg_trgm;
CREATE INDEX IF NOT EXISTS "Document_textContent_gin_idx" ON "Document" USING GIN ("textContent" gin_trgm_ops);
CREATE INDEX IF NOT EXISTS "Document_subject_gin_idx" ON "Document" USING GIN ("subject" gin_trgm_ops);
```

---

## 2. خدمة استخراج النصوص (Text Extraction Service)

عند إنشاء وثيقة، يُرسل النظام مهمتها فوراً إلى طابور عبر `BullMQ`. يعمل الـ Worker في الخلفية لاستخراج النص من كل مرفق باستخدام مكتبات متخصصة:

-   **الصور الممسوحة ضوئياً (PNG/JPEG):** تعتمد على `Tesseract.js` وحزم اللغات `ara+eng`.
-   **مستندات Word:** تعتمد على `mammoth` لاستخراج النص الخام بكفاءة.
-   **ملفات القراءة PDF:** تعتمد على `pdf-parse`.

تُجمع هذه النصوص الشاملة وتُحفظ لاحقاً في الحقل `textContent` في جدول الوثائق.

---

## 3. معالجة الواجهة الخلفية (Backend Controller)

يستقبل الـ (`document.controller.ts`) معلمة البحث. إذا كان معامل `fullText` موجوداً، يُضاف حقل `textContent` لاستعلامات البحث الفعّالة بالاعتماد على قاعدة (OR).

لتوفير الـ (Snippet)، يقوم الخادم بمعالجة الوثائق المُسترجعة كالتالي:
1. العثور على موقع الكلمة المبحوثة باستخدام دالة `indexOf`.
2. اقتطاع سلسلة نصوص تمتد بـ 60 حرفاً قبل وبعد الكلمة المبحوثة.
3. إرسال النص المقتطع (Snippet) كحقل إضافي واجهة المستعرض (Frontend).

---

## 4. عرض النتائج والتظليل (Frontend Highlights)

في صفحة النظرة العامة `Documents.tsx` نعرض الـ Snippet أسفل تفاصيل الوثيقة، ونقوم بحقن الكلمة المبحوثة داخل علامة (HTML Tag `<mark>`) لتصبح مميزة بخلفية صفراء. 
هذا النظام مساند بواجهة إضافية (Advanced Search Panel) تتيح مطابقة نطاقات التاريخ والتصنيفات والأقسام، مما يقدم تجربة شاملة ومتكاملة تماماً.
